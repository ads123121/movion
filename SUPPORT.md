# Support

## Getting help

Use the repository issue tracker for:

- reproducible bugs
- setup issues
- feature requests
- documentation gaps

When reporting a problem:

- describe the expected behavior
- describe the actual behavior
- include the OS version
- include the exact build or commit if known
- scrub any private media or personal information

## Before filing an issue

- try the latest `main`
- run `npm run build`
- run `npm run lint`
- check whether the problem is preview-only, export-only, or both
